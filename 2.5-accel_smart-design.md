# 2. 项目背景与原则

## 2.1 accel 项目的 5 大核心原则(不可违反)

### 原则 1:简单粗暴(简单是方法,不是妥协)

- 简单 = 对问题边界的清晰认识
- 拒绝伪复杂度(看起来智能但无价值的功能)
- **不为了"少代码"砍掉关键功能** ← 用户特别强调

### 原则 2:高效可靠(只展示 100% 准确数据)

- 不准的数字 = 误导用户
- 拒绝平均值这种混淆性统计
- 算法行为必须可预测

### 原则 3:大炮换弹药 + 热插拔

- 多算法并存(HashMap)
- 切换不重启,不 drop 老连接
- 用户 sysctl 切换感觉就是另一个算法

### 原则 4:不越权

- accel 只做加速,不管总带宽分配
- 不替管理员决策
- 用户付钱给 VPS 是流量,不是冗余

### 原则 5:未来 AI 调节铺路

- 配置走文件(初始值)
- 运行时 BPF map 可读写
- 当前简单实现 = 未来扩展接口

## 2.2 已实现算法

### accel_cubic(2.1)
- 等效 Linux 内核 CUBIC
- 基线算法,验证多算法架构

### accel_brutal(2.3)
- 基于 tcp-brutal 自写实现
- 80% ack_rate 钳制(brutal 灵魂)
- 单参数:`rate_mbps`(单连接速率上限)
- 适合跨境高丢包链路

### ❌ accel_bbr(2.2,放弃)
- BBR 内核源码 1199 行 + 69 缺失符号
- 用户洞察:用户可直接 sysctl bbr,accel 复刻无价值
- WIP commit `7133987` 保留历史

- 锐速/ZetaTCP/LotServer 是闭源商业产品(AppEx Networks)
- 无可信开源参考

- ZetaTCP 用了 10 年调状态机参数,我们做不了
- BPF struct_ops 实现复杂学习模型困难

# 3. accel_smart 核心思想

## 3.1 设计理念

**状态自适应 + 多倍发包冗余**,借鉴 ZetaTCP 思想,用最科学的方法实现:

```
链路状态        →   算法行为              →   tc 行为
GOOD(链路好)   →   brutal 激进发包       →   不复制(0%)
NOISY(链路噪音)→   reno 标准(让 TCP)    →   多倍发包(100% = 2倍)
CONGESTED(真拥塞)→  退让 30%             →   不复制(0%)
```

## 3.2 关键洞察(来自用户)

**STATE_NOISY 时不能用 brutal 激进 + 多倍发包**:

```
brutal:按 rate 发(已填满链路)
+ 多倍发包(每包发 2 份)
= 实际流量 2x rate
= 打爆链路
= 真拥塞产生
= 雪球越滚越大
```

**正确做法**:
```
STATE_NOISY 时让 TCP 自然退让(reno),多倍发包补偿丢包
总流量 ≈ 链路容量,冗余补丢包
```

## 3.3 这是真正的"新算法",不是切换

**用户视角**:
- 选 accel_smart
- 内部根据链路状态自动切换行为
- 不需要管细节

**内部实现**:
- 一个 BPF struct_ops(smart_main)
- 三种行为分支(brutal / reno / 退让)
- 用户态协调器同步 tc

## 4.2 状态机定义

### 状态枚举

```c
enum LinkState {
    STATE_GOOD       = 0,   // 链路好(无丢包,RTT 稳)
    STATE_NOISY      = 1,   // 链路噪音(有丢包,RTT 稳)
    STATE_CONGESTED  = 2,   // 真拥塞(RTT 飙升)
    STATE_KEEP       = 99,  // 内部:维持当前状态(滞后区)
};
```

### 状态判断阈值(精度:basis points,1% = 100 bp)

```c
/* 丢包率阈值(滞后区设计)*/
#define LOSS_GOOD_MAX_BP        50      // < 0.5% = GOOD 候选
#define LOSS_NOISY_ENTER_BP     100     // > 1.0% = NOISY 候选(进入)
#define LOSS_NOISY_EXIT_BP      50      // < 0.5% = NOISY 退出(滞后)
#define LOSS_CONGEST_BP         1500    // > 15% = CONGESTED 候选

/* RTT 阈值(相对值)*/
#define RTT_INFLATION_LOW       20      // (srtt-minrtt)/minrtt < 20% = OK
#define RTT_INFLATION_HIGH      50      // > 50% = 拥塞嫌疑
#define RTT_INFLATION_SEVERE    100     // > 100% = 严重拥塞

/* 确认次数(状态切换防抖)*/
#define CONFIRM_TO_CONGESTED    3       // 切到危险状态:快(安全)
#define CONFIRM_TO_NOISY        5       // 中间状态:中等
#define CONFIRM_TO_GOOD         10      // 切到优良状态:严格(避免错估)
```

### 状态转换规则

```c
__u32 evaluate_state(struct measurement *m) {
    /* 五个原始信号 */
    bool sig_loss_severe   = m->loss_rate_bp >= LOSS_CONGEST_BP;
    bool sig_loss_medium   = m->loss_rate_bp >= LOSS_NOISY_ENTER_BP;
    bool sig_rtt_severe    = m->rtt_inflation_pct >= RTT_INFLATION_SEVERE;
    bool sig_rtt_inflated  = m->rtt_inflation_pct >= RTT_INFLATION_HIGH;
    bool sig_retrans_high  = m->retrans_rate_bp >= 500;  // 5%+
    
    /* 单个极强信号 → CONGESTED(安全优先)*/
    if (sig_loss_severe || sig_rtt_severe) {
        return STATE_CONGESTED;
    }
    
    /* 任意两个强信号 → CONGESTED */
    int strong_signals = 0;
    if (sig_loss_medium)   strong_signals++;
    if (sig_rtt_inflated)  strong_signals++;
    if (sig_retrans_high)  strong_signals++;
    if (strong_signals >= 2) {
        return STATE_CONGESTED;
    }
    
    /* GOOD:所有差信号都不存在 */
    if (m->loss_rate_bp <= LOSS_GOOD_MAX_BP &&
        m->rtt_inflation_pct < RTT_INFLATION_LOW) {
        return STATE_GOOD;
    }
    
    /* NOISY:有丢包但 RTT 稳定(链路噪音特征)*/
    if (sig_loss_medium && !sig_rtt_inflated && !sig_retrans_high) {
        return STATE_NOISY;
    }
    
    /* 边界:维持当前(滞后)*/
    return STATE_KEEP;
}
```

### 状态确认机制

```c
struct state_confirm {
    __u32 candidate;        // 候选状态
    __u32 streak;           // 连续命中次数
};

void update_state(struct state_confirm *sc, __u32 *current, __u32 want) {
    if (want == STATE_KEEP) {
        sc->streak = 0;
        sc->candidate = *current;
        return;
    }
    
    /* 不同目标状态需要不同确认次数 */
    __u32 required;
    if (want == STATE_CONGESTED)  required = CONFIRM_TO_CONGESTED;
    else if (want == STATE_GOOD)  required = CONFIRM_TO_GOOD;
    else                           required = CONFIRM_TO_NOISY;
    
    if (want == sc->candidate) {
        sc->streak++;
        if (sc->streak >= required) {
            *current = sc->candidate;
            sc->streak = 0;
        }
    } else {
        sc->candidate = want;
        sc->streak = 1;
    }
}
```

## 4.3 测量层(滑动窗口)

### 数据结构

```c
struct sliding_window {
    /* 10 秒窗口,1 秒一槽 */
    __u32 acked[10];
    __u32 losses[10];
    __u32 retrans[10];
    __u64 bytes_acked[10];
    
    __u32 current_slot;
    __u64 last_slot_us;
};

struct measurement {
    /* 一级指标(直接读 sock)*/
    __u32 srtt_us;          // tp->srtt_us >> 3
    __u32 min_rtt_us;       // tcp_min_rtt(tp)
    __u32 mdev_us;          // tp->mdev_us >> 2
    
    /* 二级指标(滑动窗口算)*/
    __u32 loss_rate_bp;     // basis points (0-10000)
    __u32 throughput_bps;
    __u32 retrans_rate_bp;
    
    /* 三级指标(衍生)*/
    __u32 rtt_inflation_pct;  // (srtt-minrtt)/minrtt * 100
    __u32 rtt_variation_pct;  // mdev/srtt * 100
};
```

### 算法

```c
/* 每次 ACK 调用 */
void update_window(struct sliding_window *w, __u32 acked, __u32 losses,
                    __u32 retrans, __u64 bytes, __u64 now_us) {
    /* 切槽?(每秒一次)*/
    if (now_us - w->last_slot_us >= 1000000) {
        w->current_slot = (w->current_slot + 1) % 10;
        w->acked[w->current_slot] = 0;
        w->losses[w->current_slot] = 0;
        w->retrans[w->current_slot] = 0;
        w->bytes_acked[w->current_slot] = 0;
        w->last_slot_us = now_us;
    }
    
    /* 累加到当前槽 */
    w->acked[w->current_slot] += acked;
    w->losses[w->current_slot] += losses;
    w->retrans[w->current_slot] += retrans;
    w->bytes_acked[w->current_slot] += bytes;
}

/* 计算 10 秒指标 */
void compute_metrics(struct sliding_window *w, struct measurement *m) {
    __u32 total_acked = 0;
    __u32 total_losses = 0;
    __u32 total_retrans = 0;
    __u64 total_bytes = 0;
    
    #pragma unroll
    for (int i = 0; i < 10; i++) {
        total_acked   += w->acked[i];
        total_losses  += w->losses[i];
        total_retrans += w->retrans[i];
        total_bytes   += w->bytes_acked[i];
    }
    
    __u32 total = total_acked + total_losses;
    if (total > 0) {
        m->loss_rate_bp = (total_losses * 10000) / total;
    }
    if (total_acked > 0) {
        m->retrans_rate_bp = (total_retrans * 10000) / total_acked;
    }
    /* 10 秒吞吐 */
    m->throughput_bps = total_bytes * 8 / 10;
    
    /* RTT 衍生 */
    if (m->min_rtt_us > 0) {
        m->rtt_inflation_pct = ((m->srtt_us - m->min_rtt_us) * 100) / m->min_rtt_us;
    }
    if (m->srtt_us > 0) {
        m->rtt_variation_pct = (m->mdev_us * 100) / m->srtt_us;
    }
}
```

## 4.4 执行层(三种行为)

```c
__u32 apply_action(struct sock *sk, struct rate_sample *rs, __u32 state) {
    if (state == STATE_GOOD) {
        /* brutal 激进:复用 accel_brutal 现有逻辑 */
        return apply_brutal(sk, rs);
    }
    
    if (state == STATE_NOISY) {
        /* reno 标准行为:让 TCP 自然退让 */
        /* tc 层会做多倍发包补偿 */
        tcp_reno_cong_avoid(sk, ack, rs->acked_sacked);
        /* 关键:不主动设 sk_pacing_rate */
        return 0;
    }
    
    /* STATE_CONGESTED:主动退让 */
    struct tcp_sock *tp = tcp_sk(sk);
    __u32 new_cwnd = (tp->snd_cwnd * 7) / 10;
    if (new_cwnd < 2) new_cwnd = 2;
    tp->snd_cwnd = new_cwnd;
    /* 不设 pacing_rate,kernel 自然处理 */
    return 0;
}
```

## 4.5 Rust 协调器

```rust
pub struct Coordinator {
    bpf: Arc<BpfState>,
    tc_manager: TcManager,
    
    current_applied_state: LinkState,
    last_apply_time: Instant,
    
    consecutive_errors: u32,
    last_successful_apply: Instant,
    degraded_mode: bool,
}

impl Coordinator {
    pub fn tick(&mut self) -> Result<()> {
        /* 健康检查 */
        self.check_tc_alive()?;
        
        /* 读 BPF 状态 */
        let bpf_state = self.bpf.read_link_state()?;
        
        if bpf_state == self.current_applied_state {
            return Ok(());  // 无变化
        }
        
        /* 抖动保护 */
        if self.last_apply_time.elapsed() < Duration::from_secs(2) {
            return Ok(());
        }
        
        /* 应用 tc 变更 */
        let dup_pct = match bpf_state {
            STATE_GOOD       => 0,
            STATE_NOISY      => 100,
            STATE_CONGESTED  => 0,
            _                => 0,
        };
        
        match self.tc_manager.change_duplicate(dup_pct) {
            Ok(_) => {
                self.current_applied_state = bpf_state;
                self.last_apply_time = Instant::now();
                self.last_successful_apply = Instant::now();
                self.consecutive_errors = 0;
                
                if self.degraded_mode {
                    info!("协调器恢复正常");
                    self.degraded_mode = false;
                }
            }
            Err(e) => {
                self.consecutive_errors += 1;
                if self.consecutive_errors >= 5 {
                    warn!("tc 协调连续失败,降级到 brutal-only: {}", e);
                    self.tc_manager.cleanup()?;
                    self.degraded_mode = true;
                }
            }
        }
        
        Ok(())
    }
    
    fn check_tc_alive(&self) -> Result<()> {
        if !self.tc_manager.qdisc_exists()? {
            warn!("tc qdisc 不存在,重建");
            self.tc_manager.install()?;
        }
        Ok(())
    }
}
```

## 5.4 用户态读 BPF 状态

```rust
fn read_link_state(&self) -> Result<LinkState> {
    let key: u32 = 0;
    let mut all_states = Vec::new();
    
    /* per-CPU map 需要遍历所有 CPU */
    for cpu_id in 0..num_cpus::get() {
        let bytes = self.bpf.smart_state_map.lookup_percpu(...)?;
        let state: SmartState = parse(bytes);
        all_states.push(state.current_state);
    }
    
    /* 取多数派(因为是 per-CPU,可能不同 CPU 略有差异)*/
    let mut counts = [0u32; 3];
    for s in &all_states {
        counts[*s as usize] += 1;
    }
    
    /* 优先 CONGESTED(安全)*/
    if counts[STATE_CONGESTED as usize] > 0 {
        return Ok(LinkState::Congested);
    }
    if counts[STATE_NOISY as usize] >= counts[STATE_GOOD as usize] {
        Ok(LinkState::Noisy)
    } else {
        Ok(LinkState::Good)
    }
}
```

# 9. 关键决策与避坑

## 9.1 设计决策汇总

| 决策点 | 选择 | 理由 |
|---|---|---|
| 状态判断粒度 | basis points (10000) | 整数运算精度高,无浮点误差 |
| 状态防抖 | 候选状态 + 确认次数 | 比时间窗简单,BPF 友好 |
| 确认次数 | 差异化(3/5/10) | 安全优先,GOOD 严格 |
| 滑动窗口 | 10 秒 / 1 秒槽 | 工业经验值 |
| BPF map 类型 | per-CPU + per-socket storage | 无锁,高性能 |
| STATE_NOISY 行为 | reno kfunc(让 TCP)| 关键!不能用 brutal+dup 双激进 |
| 协调器频率 | 1 秒 tick | 足够响应,开销小 |
| 多倍发包工具 | tc netem duplicate | 内核原生,无新 BPF 代码 |

## 9.2 避坑清单

### 坑 1:**STATE_NOISY 不能用 brutal**

**用户的关键洞察**:brutal + 多倍发包 = 双重激进 = 打爆链路。

正确做法:NOISY 时调 `tcp_reno_cong_avoid` kfunc,让 TCP 自然行为。

### 坑 2:**单点 RTT 不能判拥塞**

srtt_us 是平滑值,变化慢。真拥塞可能已持续几秒才反映。

正确做法:用 `rtt_inflation_pct = (srtt - minrtt) / minrtt * 100`,看相对变化。

### 坑 3:**单次 ACK 丢包率无意义**

一次 ACK 可能 0% 或 50%,波动剧烈。

正确做法:10 秒滑动窗口,累积 acked 和 losses 计算 loss_rate_bp。

### 坑 4:**确认次数全相同会出 bug**

如果 GOOD/NOISY/CONGESTED 都用同样确认次数:
- 切到 CONGESTED 慢 → 链路真坏了 还在激进发包
- 切到 GOOD 快 → 偶尔好了一下就放飞,实际还是噪音

正确做法:CONGESTED 快(安全),GOOD 慢(避免错估)。

### 坑 5:**tc 命令是阻塞的**

tc 命令通过 netlink 调用,耗时 1-10ms。

正确做法:协调器在独立线程,主线程不阻塞。

### 坑 6:**协调器 panic 不能让 accel 挂**

如果协调器线程 panic,brutal 仍应工作(降级到 brutal-only)。

正确做法:协调器有 `degraded_mode` 标志,失败 5 次自动降级。

### 坑 7:**多倍发包对 SACK 的影响**

接收端可能误判为乱序 / 重复,触发 spurious retransmission。

需要在 PoC 阶段验证(看 ss 输出的 retrans 数字)。

## 9.3 失败回退机制

```
正常运行
   ↓
tc 命令失败
   ↓
连续 5 次失败
   ↓
进入降级模式:
  - tc 配置全部清理
  - 协调器停止 tick
  - brutal 算法仍工作
  - status 显示 "degraded: tc unavailable"
   ↓
每分钟尝试恢复一次
   ↓
恢复成功 → 退出降级模式
恢复失败 → 维持降级
```
